# 如何高效阅读 LLM 论文

LLM 领域的论文数量爆炸式增长，高效阅读成为必备技能。LLM Primer 的三档阅读系统同样适用于论文阅读：

## 第一遍：直觉扫描（5 分钟）

不要从头到尾读。先看标题、摘要和结论，问自己三个问题：
- 这篇论文解决什么问题？
- 核心方法是什么？一句话能否概括？
- 主要结论和局限性是什么？

如果与你的研究方向或工作无关，到此为止。

## 第二遍：工程拆解（20 分钟）

如果论文相关，进入第二遍：
- 看方法部分的流程图和伪代码，理解输入输出。
- 看实验设置：数据集、基线、评估指标。这些决定了结论的可信度。
- 记录关键数字：准确率提升了多少？计算成本增加了多少？

一个可运行的记录模板如下：

```python
paper_note = {
    "problem": "reduce inference cost",
    "method": "cache reusable key/value tensors",
    "metric": {"latency_improvement": 0.35, "memory_gb": 4.2},
}

print(f"方法：{paper_note['method']}")
print(f"延迟改进：{paper_note['metric']['latency_improvement']:.0%}")
```

## 第三遍：研究深挖（按需）

如果你要做相关研究或实现：
- 逐行推导核心公式，检查假设是否合理。
- 复现关键实验，验证论文声称的结果。
- 思考扩展方向：这个方法能否应用到其他任务？局限性能否被克服？

读实验结果时，可以把收益和成本放在同一条公式里：

$$
\text{efficiency} = \frac{\Delta \text{quality}}{\Delta \text{compute}}
$$

## 利用 LLM Primer 的论文库

每篇收录论文都附有双语 TLDR，帮你快速完成第一遍扫描。点击论文条目，还能看到哪些文章引用了它，从而了解该论文在知识体系中的位置。

记住：读论文的目的不是"读完"，而是"提取对你有用的信息"。
